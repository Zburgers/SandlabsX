# OSPF failure recovery Scenario

This Scenario is learning content separate from the Capsule topology. Before publishing it, replace `PIN_THE_PUBLISHED_CAPSULE_VERSION` with the immutable ID of the compatible OSPF Capsule version.

The Scenario expects an isolated, student-owned instance with router `r1`. It first confirms the router is running, then checks bounded serial evidence for an OSPF `FULL` adjacency. It does not require a proprietary image and contains no executable host code. Any optional guest command check must target an instance-owned VM and use the qualified guest transport's argument-array interface.
